# 技术组数据库 - 部署与分享指南

## 📋 前置准备

### 1. 确保Supabase配置正确

检查 `.env` 文件中的配置：

```env
VITE_SUPABASE_URL=https://your-project.supabase.co
VITE_SUPABASE_ANON_KEY=your-anon-key
```

**⚠️ 重要提醒：**
- 不要将 `.env` 文件提交到代码仓库
- 使用 `.env.example` 作为模板
- 为每个部署环境创建独立的Supabase项目或配置

### 2. 确保数据库表已创建

在Supabase SQL编辑器中执行以下迁移（按顺序）：

1. `supabase/migrations/00001_create_categories_and_files_tables.sql`
2. `supabase/migrations/00002_create_storage_buckets.sql`
3. `supabase/migrations/00003_update_to_infinite_categories.sql`
4. `supabase/migrations/00004_add_specification_column.sql`

## 🚀 方法一：本地局域网分享（推荐用于内部小团队）

### 步骤1：构建生产版本

```bash
npm install
npm run build
```

### 步骤2：启动预览服务器

```bash
npm run preview
```

### 步骤3：分享给同事

1. 查看您的本地IP地址：
   - Windows: 在命令行输入 `ipconfig`，找到 IPv4 地址
   - 例如：`192.168.1.100`

2. 修改 `vite.config.ts`，添加host配置：
```typescript
export default defineConfig({
  server: {
    host: '0.0.0.0',
    port: 4173,
  },
  // ...其他配置
})
```

3. 分享链接给同事：
   - `http://您的IP地址:4173`
   - 例如：`http://192.168.1.100:4173`

### 注意事项
- 确保防火墙允许端口访问
- 确保所有用户都连接到同一局域网
- 所有用户共享同一个Supabase数据库

## 🌐 方法二：部署到Vercel（推荐用于公开访问）

### 步骤1：准备代码仓库

1. 确保 `.gitignore` 包含：
```
.env
.env.local
.env.*.local
node_modules
dist
```

2. 提交代码到Git仓库（GitHub/GitLab等）

### 步骤2：Vercel部署

1. 访问 https://vercel.com 并登录
2. 点击 "New Project"
3. 导入您的Git仓库
4. 在环境变量配置中添加：
   - `VITE_SUPABASE_URL` = 您的Supabase URL
   - `VITE_SUPABASE_ANON_KEY` = 您的Supabase Anon Key
5. 点击 "Deploy"

### 步骤3：配置Supabase

在Supabase项目设置中添加Vercel域名到CORS白名单：
- Authentication -> URL Configuration
- 添加您的Vercel域名，例如：`https://your-app.vercel.app`

## 💾 方法四：使用Docker部署

### 创建Dockerfile

在项目根目录创建 `Dockerfile`：

```dockerfile
FROM node:18-alpine as builder
WORKDIR /app
COPY package*.json ./
RUN npm install
COPY . .
RUN npm run build

FROM nginx:alpine
COPY --from=builder /app/dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/nginx.conf
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]
```

### 创建nginx.conf

```nginx
events {}
http {
  include /etc/nginx/mime.types;
  server {
    listen 80;
    server_name localhost;
    root /usr/share/nginx/html;
    index index.html;
    location / {
      try_files $uri $uri/ /index.html;
    }
  }
}
```

### 构建和运行

```bash
docker build -t tech-database .
docker run -p 80:80 tech-database
```

## 🔐 安全建议

### 1. 数据库安全
- 使用Supabase的行级安全策略（RLS）
- 不要使用service_role_key在前端
- 定期轮换anon_key

### 2. 环境变量
- 永远不要提交.env文件到代码仓库
- 使用环境变量管理工具
- 为不同环境使用不同的Supabase项目

### 3. 文件存储
- 配置适当的存储策略
- 设置文件大小限制
- 定期备份重要文件

## 📱 移动端访问

应用已支持响应式设计，可以直接在手机浏览器中访问。

## 🔧 故障排除

### 问题：页面空白
- 检查浏览器控制台错误
- 确认Supabase URL和Key配置正确
- 检查网络连接

### 问题：无法上传文件
- 确认Supabase存储桶已创建
- 检查存储策略设置
- 检查文件大小限制

### 问题：CORS错误
- 在Supabase中添加域名到CORS白名单
- 检查API URL配置
