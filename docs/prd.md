# 需求文档

## 1. 应用概述

### 1.1 应用名称
文件快速查找工具

### 1.2 应用描述
一款支持文件分类管理与快速查找的工具应用。用户可上传文件的名称、图片、源文件等信息，系统按用户设定的二级分类结构进行归档，支持分类管理、关键词搜索及文件信息查看，所有用户均可访问查看。

## 2. 页面结构与功能说明

### 2.1 页面结构

```
文件快速查找工具
├── 首页（文件列表页）
├── 分类浏览页
├── 文件上传页
└── 分类管理页
```

### 2.2 首页（文件列表页）

**功能说明：**
- 展示所有已上传的文件列表
- 每个文件显示：文件名称、缩略图、上传时间、上次修改时间
- 提供关键词搜索框，支持按文件名称搜索
- 提供「按分类浏览」入口，跳转至分类浏览页
- 提供「上传文件」入口，跳转至文件上传页
- 提供「分类管理」入口，跳转至分类管理页
- 点击文件可查看详情（名称、图片、源文件、上传时间、上次修改时间）

### 2.3 分类浏览页

**功能说明：**
- 展示所有一级分类列表
- 点击一级分类，展开显示该分类下的二级分类列表
- 点击二级分类，展示该分类下的所有文件
- 文件展示内容同首页文件列表

### 2.4 文件上传页

**功能说明：**
- 填写文件名称
- 上传图片
- 上传源文件
- 选择一级分类与二级分类（下拉选择）
- 提交后，文件信息保存至对应分类，并记录上传时间
- 上传成功后返回首页

### 2.5 分类管理页

**功能说明：**
- 展示所有一级分类与二级分类的树状结构
- 支持新增一级分类
- 支持在一级分类下新增二级分类
- 支持删除一级分类（删除前需确认，删除后该分类下的二级分类及文件一并删除）
- 支持删除二级分类（删除前需确认，删除后该分类下的文件一并删除）

## 3. 业务规则与逻辑

### 3.1 分类规则
- 分类结构为二级：一级分类 → 二级分类
- 每个文件必须归属于一个二级分类
- 一级分类下必须至少有一个二级分类才能上传文件

### 3.2 搜索规则
- 关键词搜索仅匹配文件名称
- 搜索结果按上传时间倒序排列

### 3.3 时间记录规则
- 上传时间：文件首次上传时自动记录
- 上次修改时间：文件信息被编辑或重新上传时更新

## 4. 异常与边界情况

| 场景 | 处理方式 |
|------|----------|
| 搜索无结果 | 提示「未找到相关文件」 |
| 分类下无文件 | 显示「该分类暂无文件」 |
| 删除分类时该分类下有文件 | 弹窗确认「删除后该分类下的所有文件将一并删除，是否继续？」 |
| 上传文件时未选择分类 | 提示「请选择分类后再上传」 |
| 删除一级分类时该分类下有二级分类 | 弹窗确认「删除后该分类下的所有二级分类及文件将一并删除，是否继续？」 |

## 5. 验收标准

1. 用户可成功上传文件（名称、图片、源文件），并选择对应的一级分类与二级分类
2. 文件列表正确显示文件名称、缩略图、上传时间、上次修改时间
3. 关键词搜索功能正常，可按文件名称查找文件
4. 分类浏览功能正常，可按一级分类与二级分类查看文件
5. 分类管理功能正常，可新增、删除一级分类与二级分类
6. 删除分类时，该分类下的文件一并删除
7. 所有用户无需登录即可访问查看所有文件
