在项目根目录运行：

```bash

```

```bash

git rm --cached .env
```

✅ **正确做法：**
1. 在Vercel项目设置 → Environment Variables 中添加
2. 直接在Vercel界面输入 `VITE_SUPABASE_URL` 和 `VITE_SUPABASE_ANON_KEY`
3. 不要在代码中提交这些值

## 🎯 如果需要更换密钥（推荐）

### 步骤1：在Supabase中生成新密钥

1. 访问 https://supabase.com/dashboard
2. 选择您的项目
3. 进入 Settings → API
4. 可以随时重新生成 `anon key`

### 步骤2：更新本地.env

```env
VITE_SUPABASE_URL=https://your-project.supabase.co
VITE_SUPABASE_ANON_KEY=your-new-anon-key
```

### 步骤3：更新Vercel环境变量（如果已部署）

在Vercel项目设置中更新环境变量，然后重新部署。

## ❓ 常见问题

### Q: 别人拿到了我的anon key会怎样？

A: anon key设计上就是公开的，它只能做您的RLS策略允许的事情。但为了安全，建议：
1. 确保RLS策略配置正确
2. 不要分享service_role_key（这个是密钥！）
3. 如果担心，就重新生成anon key

### Q: 我可以把项目推送到公开GitHub仓库吗？

A: 可以！只要确保：
- ✅ .env 在 .gitignore 中
- ✅ 只有 .env.example 在仓库中
- ✅ 没有密钥硬编码在代码中

### Q: 团队协作怎么处理密钥？

A: 推荐方式：
1. 使用密码管理工具共享（如1Password）
2. 或者每人使用自己的Supabase项目
3. 或者使用环境变量管理服务
